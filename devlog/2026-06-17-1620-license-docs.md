# License documentation expansion

## What landed

- Replaced the short CC BY-SA summary in `LICENSE` with the full
  CC BY-SA 4.0 legal text.
- Added `LICENSE-PHILOSOPHY.md` and linked it from `README.md`.
- Normalized the earlier license-change devlog filename to the
  timestamped convention.

## Decisions

- **Full legal text in `LICENSE`.** This supersedes the earlier choice
  to keep only a human-readable summary. Keeping the complete license in
  the repository avoids making `LICENSE` depend on a paraphrase or a
  remote page for the operative terms.
- **Philosophy stays separate from the README.** The README should name
  the license and link to rationale without becoming an essay; the
  philosophy document can explain the broader license model for skills,
  libraries, and applications.
- **Keep CC BY-SA for prompt skills.** The added rationale preserves the
  prior decision that skills are knowledge artifacts and should remain
  share-alike rather than merely attribution-only or non-commercial.

## Deferred

- No separate license-header convention for individual skill files yet.
