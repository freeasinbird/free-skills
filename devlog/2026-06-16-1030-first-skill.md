# First skill: agent-setup

## What landed

- Added `skills/agent-setup/` — the agent-setup skill ported from the
  personal Claude Code skills directory into the open repo format.
- Added `.markdownlint.yaml` and `.gitignore` as project infrastructure.

## Decisions

- **Prompt is a direct copy of the source skill.** No changes to prompt
  logic or behavior — this is a straight port to establish the pattern.
  The skill references `references/` files via relative paths, which
  work in both locations.
- **Disabled MD013 (line length), MD041 (first-line heading), MD060
  (table column style)** in markdownlint config. Line length is
  impractical for prose-heavy markdown; MD041 conflicts with the PR
  template (starts with a comment); MD060 rejects padded tables that
  are more readable.
- **Prettier used for auto-formatting** — it touched AGENTS.md, prompt.md,
  and canonical-sections.md (blank lines around HTML comments, minor
  table alignment). Accepted all changes.

- CI configuration — no `.github/workflows/` yet. Flagged in AGENTS.md

- Conventions section — left as placeholder until patterns emerge from
