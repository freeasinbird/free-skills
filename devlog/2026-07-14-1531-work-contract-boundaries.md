# Establish work contracts and assignment boundaries

This direct, one-session user assignment needs no tracker issue. The prompt,
this entry, and the eventual PR collectively carry its work contract.

## Work contract

- **Objective:** add lightweight work contracts and explicit assignment and
  concurrency boundaries to agent-setup.
- **Acceptance criteria:** the seven user-supplied behavioral cases read
  unambiguously, the managed copies stay identical, the full requested check
  battery passes, and the PR reaches review-ready handoff.
- **Scope:** the canonical finish-line and branches text, agent-setup's
  project-specific init guidance, this repo's synchronized AGENTS.md, and this
  entry.
- **Dependencies and blockers:** the established fresh-base, worktree, stack,
  and review-watch rules remain unchanged; free-skills has no selected
  forge-visible claim mechanism.
- **Non-goals:** no issue template, claim syntax, label taxonomy, autonomous
  pickup, scheduling apparatus, path enforcement, other skill changes, or
  Freeside-specific machinery.

## Decisions

- Chose the finish line for the universal contract and authorization defaults,
  with branches carrying only dependency breakdown and claim occupancy. This
  avoids a new managed section and keeps each rule in one canonical home.
- Preserved the user-confirmed rejection of a universal issue template and
  claim protocol. agent-setup asks each project to choose its own mechanism.
- Free-skills has no forge-visible claim convention. This remains an explicit
  unresolved project choice; this work does not invent one.
